## 1. 프로젝트 개요

### 1.1 목표

사용자별 할일 관리 및 공휴일 통합 관리 웹 애플리케이션 구축

### 1.2 전체 일정

- **총 예상 기간**: 7-11주 (병렬 작업 시 7-9주 가능)
- **Phase 1 (MVP - P0)**: 4-6주
- **Phase 2 (Enhancement - P1)**: 2-3주
- **Phase 3 (Polish - P2)**: 1-2주

### 1.3 마일스톤

| 마일스톤             | 완료 기준                             | 예상 완료일 |
| -------------------- | ------------------------------------- | ----------- |
| M1: 개발 환경 구축   | DB, 백엔드, 프론트엔드 초기 설정 완료 | Week 1      |
| M2: 인증 시스템 완료 | 회원가입, 로그인, 토큰 관리 동작      | Week 2      |
| M3: 할일 CRUD 완료   | 할일 생성, 조회, 수정, 삭제 동작      | Week 4      |
| M4: 휴지통 기능 완료 | 휴지통 조회, 복원, 영구삭제 동작      | Week 5      |
| M5: MVP 배포         | P0 기능 모두 완료 및 프로덕션 배포    | Week 6      |
| M6: P1 기능 완료     | 공휴일, 토큰 갱신, 반응형 완료        | Week 8      |
| M7: 최종 배포        | 모든 기능 완료 및 성능 최적화         | Week 9-11   |

| API 성능 저하          | 중     | 중     | 쿼리 최적화, 인덱스 활용, 캐싱, 성능 테스트    | Backend  |
| 프론트엔드 성능 이슈   | 중     | 중     | 코드 스플리팅, Lazy Loading, 번들 최적화       | Frontend |
| 크로스 브라우저 호환성 | 중     | 중     | 조기 테스트, Polyfill 사용                     | Frontend |
| 의존성 취약점          | 중     | 중     | 정기 업데이트, npm audit, Dependabot           | All      |
| 타입 안전성 이슈       | 낮음   | 낮음   | TypeScript strict 모드, 타입 검증              | All      |

### 6.2 일정 리스크

| 리스크               | 가능성 | 영향도 | 완화 전략                                      | 담당   |
| -------------------- | ------ | ------ | ---------------------------------------------- | ------ |
| 요구사항 변경        | 중     | 중     | MVP 범위 명확화, 변경 관리 프로세스            | PM     |
| 기술 학습 곡선       | 낮음   | 중     | 사전 학습, 문서화, 페어 프로그래밍             | All    |
| 통합 이슈            | 중     | 중     | 조기 통합 테스트, API 명세 준수, 일일 스탠드업 | All    |
| 테스트 커버리지 부족 | 중     | 중     | TDD 적용, 테스트 우선 작성, 코드 리뷰          | All    |
| 배포 지연            | 낮음   | 높음   | CI/CD 조기 구축, 스테이징 환경 테스트          | DevOps |

| -------------------- | ------ | ------ | --------------------------------------- | ----- |

4. **완화 실행**: 리스크 발생 시 완화 전략 즉시 실행

## 10. 일일 스탠드업 및 주간 리뷰

### 일일 스탠드업 (15분)

- **시간**: 매일 오전 10시
- **형식**:
  1. 어제 완료한 작업
  2. 오늘 계획한 작업
  3. 블로커 및 도움 필요 사항

### 주간 리뷰 (1시간)

- **시간**: 매주 금요일 오후 3시
- **안건**:
  1. 주간 완료 Task 검토
  2. 다음 주 계획
  3. 리스크 검토
  4. 마일스톤 진행률 확인

### Sprint 계획 (2주 단위)

- **Sprint 1 (Week 1-2)**: 환경 구축 + 인증
- **Sprint 2 (Week 3-4)**: 할일 CRUD
- **Sprint 3 (Week 5-6)**: 휴지통 + MVP 배포
- **Sprint 4 (Week 7-8)**: P1 기능
- **Sprint 5 (Week 9-10)**: P2 기능 (선택)
- **Sprint 6 (Week 11)**: 최종 정리 및 배포 (선택)
