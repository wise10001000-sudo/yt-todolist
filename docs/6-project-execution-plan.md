## 1. 프로젝트 개요

### 1.1 프로젝트 목표

개인별 할일을 효율적으로 관리하고, 국경일 등 공통 일정과 함께 통합 관리할 수 있는 직관적이고 신뢰할 수 있는 웹 기반 할일 관리 플랫폼 구축

### 1.2 전체 일정

- **총 기간:** 7-11주
- **Phase 1 (MVP):** 4-6주
- **Phase 2 (Enhancement):** 2-3주
- **Phase 3 (Polish):** 1-2주

### 1.3 주요 마일스톤

| 마일스톤 | 완료 기준 | 목표 일정 |
|---------|----------|----------|
| MS-01: 개발 환경 구축 | 프론트엔드, 백엔드, DB 개발 환경 완료 | 1주차 |
| MS-02: 인증 시스템 완료 | 회원가입, 로그인 기능 동작 | 2주차 |
| MS-03: 할일 CRUD 완료 | 할일 생성, 조회, 수정, 삭제 기능 동작 | 3주차 |
| MS-04: 휴지통 기능 완료 | 복원, 영구 삭제 기능 동작 | 4주차 |
| MS-05: MVP 배포 | Phase 1 기능 전체 프로덕션 배포 | 5-6주차 |
| MS-06: 공휴일 통합 완료 | 공휴일 조회 및 캘린더 통합 | 7-8주차 |
| MS-07: 최종 배포 | Phase 2 기능 포함 최종 배포 | 9주차 |

### 6.4 리스크 대응 절차

1. **리스크 식별**
   - 주간 회의에서 새로운 리스크 검토
   - 팀원 누구나 리스크 제기 가능

2. **리스크 평가**
   - 가능성: 낮음(1) / 중(2) / 높음(3)
   - 영향도: 낮음(1) / 중(2) / 높음(3)
   - 우선순위 = 가능성 × 영향도

3. **리스크 대응**
   - 회피: 발생 가능성 제거
   - 완화: 영향도 감소
   - 전가: 외부 서비스 활용
   - 수용: 비용 대비 효과 낮은 경우

4. **리스크 모니터링**
   - 상태 업데이트 (대기, 진행 중, 완료, 발생)
   - 주간 리스크 리뷰
   - 발생 시 즉시 에스컬레이션
