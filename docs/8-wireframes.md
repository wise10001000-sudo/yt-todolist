## 1. 와이어프레임 개요

이 문서는 `yt-todolist` 애플리케이션의 주요 화면에 대한 와이어프레임 스케치 및 화면 흐름을 정의합니다. `제품 요구사항 명세서 (3-prd.md)` 및 `사용자 시나리오 (4-user-scenarios.md)`를 기반으로 사용자 인터페이스의 레이아웃, 콘텐츠, 기능 및 상호작용을 시각화합니다.

---

## 2. 주요 화면 와이어프레임

### 2.1. 회원가입 화면 (`/register`)

중앙에 폼이 위치한 간단한 화면입니다.

```
+------------------------------------------+
|                                          |
|                yt-todolist               |
|                                          |
| +--------------------------------------+ |
| |        이메일을 입력하세요             | |
| +--------------------------------------+ |
|                                          |
| +--------------------------------------+ |
| |        비밀번호 (8자 이상)      [👁️] | |
| +--------------------------------------+ |
|   * 유효하지 않은 비밀번호입니다.          |
|                                          |
| +--------------------------------------+ |
| |        이름 (2-50자)                 | |
| +--------------------------------------+ |
|                                          |
| +--------------------------------------+ |
| |              회원가입                | |
| +--------------------------------------+ |
|                                          |
|          이미 계정이 있나요? 로그인         |
|                                          |
+------------------------------------------+
```

---

### 2.2. 로그인 화면 (`/login`)

회원가입 화면과 유사하지만, 입력 필드가 더 적습니다.

```
+------------------------------------------+
|                                          |
|                yt-todolist               |
|                                          |
| +--------------------------------------+ |
| |             이메일 주소              | |
| +--------------------------------------+ |
|                                          |
| +--------------------------------------+ |
| |              비밀번호           [👁️] | |
| +--------------------------------------+ |
|                                          |
| +--------------------------------------+ |
| |                로그인                | |
| +--------------------------------------+ |
|                                          |
|         계정이 없으신가요? 회원가입        |
|                                          |
+------------------------------------------+
```

---

### 2.3. 메인 대시보드 (`/dashboard`)

헤더, 필터, 할일 목록, 그리고 새 할일 추가 버튼(FAB)으로 구성됩니다.

```
+------------------------------------------------------------------+
| yt-todolist                              [홍길동님] [로그아웃] |
+------------------------------------------------------------------+
|                                                                  |
|  필터: [오늘] [이번 주] [이번 달] [전체]   정렬: [종료일순 v]     |
|                                                                  |
| +--------------------------------------------------------------+ |
| | 🎄 크리스마스                                                 | |
| |   2025-12-25                                (공휴일)         | |
| +--------------------------------------------------------------+ |
|                                                                  |
| +--------------------------------------------------------------+ |
| | 📌 프로젝트 기획서 작성                                      | |
| |   내부 검토를 위한 초안 작성...                             | |
| |   📅 2025-11-25 ~ 2025-11-30                [수정] [삭제]  | |
| +--------------------------------------------------------------+ |
|                                                                  |
| +--------------------------------------------------------------+ |
| | 📌 알고리즘 과제 제출 (만료 임박)                          | |
| |   ...                                                        | |
| |   📅 2025-11-25 ~ 2025-11-27                [수정] [삭제]  | |
| +--------------------------------------------------------------+ |
|                                                                  |
|                                                              (+) |
+------------------------------------------------------------------+
```
* **할일이 없을 경우:**
```
+------------------------------------------------------------------+
| ... (헤더 및 필터 생략) ...                                       |
|                                                                  |
|                                                                  |
|                      +----------------+                          |
|                      |      (^_^)     |                          |
|                      +----------------+                          |
|                                                                  |
|                   아직 할일이 없습니다.                           |
|                   첫 할일을 추가해보세요!                          |
|                                                                  |
|                                                              (+) |
+------------------------------------------------------------------+
```

### 2.4. 할일 추가/수정 모달

대시보드 화면 위에 오버레이로 표시됩니다.

```
+------------------------------------------------------------------+
| ........................ (배경) .................................|
| .                                                              . |
| .       +--------------------------------------------+         . |
| .       | 새 할일 추가                          [X]  |         . |
| .       +--------------------------------------------+         . |
| .       |                                            |         . |
| .       |  제목: [프로젝트 기획서 작성]                |         . |
| .       |                                            |         . |
| .       |  내용: [내부 검토를 위한 초안 작성]          |         . |
| .       |                                            |         . |
| .       |  시작일: [2025-11-25 09:00] 🗓️             |         . |
| .       |  종료일: [2025-11-30 23:59] 🗓️             |         . |
| .       |                                            |         . |
| .       |                    [취소]      [저장]      |         . |
| .       +--------------------------------------------+         . |
| .................................................................. |
+------------------------------------------------------------------+
```

---

### 2.5. 휴지통 화면 (`/trash`)

삭제된 항목들을 관리하는 화면입니다.

```
+------------------------------------------------------------------+
| [<- 뒤로가기]  휴지통                  [홍길동님] [로그아웃] |
+------------------------------------------------------------------+
|                                                                  |
| +--------------------------------------------------------------+ |
| | 팀 회의 준비                                                 | |
| |   삭제일시: 2025-11-25 20:30                [복원] [영구 삭제] | |
| +--------------------------------------------------------------+ |
|                                                                  |
| +--------------------------------------------------------------+ |
| | 영어 발표 준비                                               | |
| |   삭제일시: 2025-11-24 18:15                [복원] [영구 삭제] | |
| +--------------------------------------------------------------+ |
|                                                                  |
|                                                                  |
+------------------------------------------------------------------+
```
* **영구 삭제 확인 다이얼로그:**
```
+------------------------------------------------------------------+
| ........................ (배경) .................................|
| .                                                              . |
| .           +--------------------------------------+           . |
| .           | ⚠️  경고                             |           . |
| .           +--------------------------------------+           . |
| .           |                                      |           . |
| .           |  영구 삭제하시겠습니까?              |           . |
| .           |  이 작업은 되돌릴 수 없습니다.       |           . |
| .           |                                      |           . |
| .           |      [취소]        [영구 삭제]       |           . |
| .           +--------------------------------------+           . |
| .................................................................. |
+------------------------------------------------------------------+
```

## 3. 반응형 디자인 고려사항

`3-prd.md - 6.2 반응형 디자인`에 따라 다음 브레이크포인트를 고려하여 와이어프레임을 디자인합니다.

*   **Mobile (< 768px):** 단일 컬럼 레이아웃, 하단 네비게이션 바, 풀 스크린 모달, Floating Action Button
*   **Tablet (768px ~ 1024px):** 2컬럼 레이아웃 (선택 사항), 사이드바 네비게이션
*   **Desktop (> 1024px):** 최대 너비 제한 (1200px), 멀티 컬럼 레이아웃, 사이드바 네비게이션
